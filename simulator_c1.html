The Knowledge Base
INGEST

Every RAG system begins with a corpus — the body of documents the model can refer to. These could be PDFs, web pages, manuals, or internal wikis. Below: a small library of three documents the simulator will use as ground truth.

Documents are too large to handle whole. Each one is split into smaller chunks — overlapping passages of a few sentences each. Smaller chunks mean more precise retrieval; too-small chunks lose context.

Each chunk is passed through an embedding model, which converts text into a high-dimensional vector — a list of numbers capturing meaning. Similar ideas land near each other in this space.

SEMANTIC SPACE · 2D PROJECTION

When a user asks a question, the query is embedded the same way. The system then finds chunks whose vectors are closest to the query vector — usually by cosine similarity. The top matches become the context.

USER QUERY
—

The retrieved chunks are stuffed into the prompt as context, followed by the original question. The LLM now answers using both its training and the supplied evidence — grounded, citable, and far less likely to hallucinate.
